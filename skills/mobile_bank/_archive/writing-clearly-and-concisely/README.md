# 清晰简洁地写作

一个应用小威廉·斯特伦克（William Strunk Jr.）永恒写作原则的技能，以产生更清晰、更有力、更专业的散文，同时避免常见的AI写作模式。

## 目的

此技能帮助您为人类读者撰写更好的散文。它来自两个来源：

1. **风格的要素**（Strunk，1918年）- 清晰有力写作的久经考验的规则
2. **AI模式避免** - 基于研究的指导，避免LLM倾向于产生的通用、浮夸语言

无论您是在撰写文档、提交信息、错误消息还是任何供人类阅读的文本，此技能都能帮助您删除冗余并表达您的意思。

## 何时使用

在撰写供人类阅读的散文时使用此技能：

- **文档** - README文件、API文档、技术解释
- **Git工作流** - 提交信息、拉取请求描述
- **面向用户的文本** - 错误消息、UI文本、帮助文本、工具提示
- **代码注释** - 内联文档、文档字符串
- **报告和摘要** - 状态更新、分析、解释
- **编辑** - 改进现有文本的清晰度

**触发短语：**
- "为...编写文档"
- "起草README"
- "编辑此内容以提高清晰度"
- "使此内容更简洁"
- "审查此提交信息"

## 工作原理

1. **加载技能** 在撰写供人类阅读的散文时
2. **应用Strunk的核心原则** - 主动语态、肯定形式、具体语言、删除不必要的词
3. **避免AI模式** - 不要浮夸、不要空短语、不要促销性形容词
4. **参考详细指南** 在需要特定规则时

## 主要特性

### Strunk的核心规则

该技能强调《风格的要素》中的这些原则：

| 规则 | 原则 |
|------|-----------|
| 10 | 使用主动语态 |
| 11 | 以肯定形式陈述 |
| 12 | 使用明确、具体、具体的语言 |
| 13 | 省略不必要的词 |
| 16 | 将相关词放在一起 |
| 18 | 将强调词放在句子末尾 |

### AI模式检测

该技能识别并消除常见的LLM写作模式：

- **浮夸**：关键的、至关重要的、重要的、证明、持久的遗产
- **空的"-ing"短语**：确保可靠性、展示功能
- **促销性形容词**：开创性的、无缝的、强大的、前沿的
- **过度使用的AI词汇**：深入、利用、多方面的、培养、领域、织锦
- **格式过度使用**：过多的项目符号、表情符号装饰、每隔一个词就加粗

## 最佳实践

1. **要具体，不要浮夸** - 说出它实际做什么，而不是它有多重要
2. **先删减，后添加** - 删除词直到意思受损，然后添加回需要的内容
3. **优先使用主动语态** - "函数返回X"胜过"X由函数返回"
4. **肯定陈述** - "他忘记了"胜过"他没有记住"
5. **使用具体语言** - "服务器崩溃了"胜过"发生了问题"
6. **谨慎加载参考文件** - 大多数任务只需要 `03-elementary-principles-of-composition.md`
